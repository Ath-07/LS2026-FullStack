# LS2026 Full Stack — Learner's Space 2026

A progressive full-stack curriculum built over four weeks, starting from vanilla front-end and culminating in a full-stack hospital management system.

## Structure

| Week | Project | Stack |
|---|---|---|
| [Week 1](./Week1) | Judgmental Password Strength Meter | HTML, CSS, Vanilla JS |
| [Week 2](./Week2) | Food Menu Ordering UI | React 19, Vite 8, Tailwind CSS 4 |
| [Week 3](./Week3) | Stagecoach Manifest API | Django 6, DRF 3.17, SimpleJWT |
| [Capstone](./hospital-project) | Hospital Management System | Django REST + React SPA |

### Week 1 — Password Strength Meter
A deliberately terrible UI component that evaluates passwords in real-time with sarcastic feedback, random-color strength bars, and intentionally bad design.

### Week 2 — Food Ordering UI
A single-page food ordering app with a responsive menu grid, add-to-cart functionality, and a cart sidebar with quantity controls.

### Week 3 — Stagecoach Manifest API
A RESTful API themed around a Wild West stagecoach line. Features JWT authentication, per-user CRUD for shipments, rate limiting, caching, and pagination.

### hospital-project — Hospital Management System
A full-stack application with role-based access (patient/doctor), appointment booking, doctor management, and JWT auth. Combines a Django REST backend with a React + Vite + React Router frontend.

## Progression

1. **Week 1** — DOM manipulation with vanilla HTML/CSS/JS
2. **Week 2** — Component-based architecture with React + Tailwind
3. **Week 3** — Backend APIs with Django REST Framework
4. **hospital-project** — Full-stack integration: Django DRF backend + React SPA frontend, role-based auth, and CRUD workflows
